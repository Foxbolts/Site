A practical primer on MCP

01

So what is MCP?

MCP is a standard way for AI apps to connect to external tools and data. One protocol, any AI, any tool.

  • It's an open standard that works with any LLM: Claude, GPT, Gemini, Llama, etc.
  • The AI can then read data from those tools and take actions in them, with your permission
  • An AI app (like Codex or Claude Code) connects to MCP servers that wrap things you already use: GitHub, Slack, databases, your filesystem
  • Think of it like USB-C for AI: instead of every AI app building its own connector to every tool, MCP gives them a shared plug

MCP vs. MCP servers: MCP is the protocol, the shared language. An MCP server is a specific program that speaks that language to connect the AI to one service (like Jira or GitHub). You don't install "MCP" itself, you install MCP servers.

02

MCP vs. calling APIs directly

Direct API Calls
AI app hardcodes each API's auth, endpoints, and data format
Every new tool = new custom code in the AI app
AI doesn't know what tools exist until a developer wires them in
Tightly coupled: if the API changes, the integration breaks
Via MCP
MCP server handles the API details; AI app just speaks MCP
New tool = add a config entry, no code changes
AI automatically discovers available tools on startup
Loosely coupled: update the server, the AI app doesn't change

MCP servers are thin wrappers around existing APIs. The API still does the real work. MCP just gives the AI a universal way to find and use it.

MCP in action: a real example

01
You type your prompt: "What's the status of FXIOS-123 and who's assigned to it?" local
02
The AI picks a tool: it sees the Jira MCP server is connected and selects the get_issue tool local
03
Permission check: Codex asks you to approve the tool call before it runs local
04
AI app sends the request to the MCP server: Codex calls the Jira MCP server with the tool name and parameters (e.g. issue key: FXIOS-123) local → remote
05
MCP server calls the Jira API: it handles auth, builds the request, and fetches the ticket data remote
06
Data comes back: the server returns structured JSON (status, assignee, summary, etc.) to the AI remote → local
07
The AI distills the JSON into a natural language answer: "FXIOS-123 is In Progress, assigned to Matt. It's a bug fix for the tab bar layout on iPad." local
04

The problem MCP solves

Before MCP
ChatGPT → GitHub: custom integration
ChatGPT → Slack: custom integration
Claude → GitHub: another integration
Claude → Slack: another integration
Cursor → GitHub: yet another...
N models × M tools = N×M integrations
With MCP
Any AI Host → MCP → GitHub server
Any AI Host → MCP → Slack server
Any AI Host → MCP → Any tool
N + M integrations. Build once, use everywhere.
05

Remote MCP servers

MCP servers can be hosted by the services you already use. Your AI connects to them over the internet via a URL.

🐙

GitHub

Create PRs, search issues, review diffs, manage branches. All from a conversation with your AI.

github/github-mcp-server →

📋

Jira

Query tickets, update status, add comments, create new issues. No more tab-switching to your project board.

atlassian.com/mcp →

🎨

Figma

Pull design specs, inspect components, and reference layouts directly in your AI-powered workflow.

figma.com/mcp →

MCP servers can also run locally on your machine for things like filesystem access, local databases, or custom scripts. Your AI app can connect to both local and remote servers at the same time.

06

What a config looks like

A real MCP config from Codex, with remote servers for Figma, Jira, and GitHub:

"mcp_servers": {
  "figma": {
    "url": "https://mcp.figma.com/mcp"
  },
  "jira": {
    "enabled": true,
    "url": "https://mcp.atlassian.com/v1/mcp"
  },
  "github": {
    "bearer_token_env_var": "GITHUB_TOKEN",
    "enabled": true,
    "url": "https://api.github.com/copilot/mcp/"
  }
}

Each entry names a server and provides its URL. Some include auth tokens via environment variables. Others (like Figma and Jira) use OAuth, so they'll prompt you to sign in on first use. That's it: add the entry, and your AI can start using that service.
